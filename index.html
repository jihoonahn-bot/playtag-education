API 설정 방법
⚡ 빠른 설정 — Client ID 직접 입력

* 적용 후 페이지가 새로고침됩니다. gcal.js 파일의 GCAL_CLIENT_ID 값도 동일하게 수정해주세요.

1
Google Cloud Console 프로젝트 생성
console.cloud.google.com 접속 → 상단 프로젝트 선택 → 새 프로젝트 → 이름 입력 (예: playtag-scheduler) → 만들기
2
Google Calendar API 활성화
왼쪽 메뉴 API 및 서비스 → 라이브러리 → 검색창에 "Google Calendar API" 입력 → 클릭 후 사용 설정 버튼 클릭
3
OAuth 동의 화면 설정
API 및 서비스 → OAuth 동의 화면 → 외부 선택 → 앱 이름(PlayTag), 이메일 입력 → 범위에서 .../auth/calendar 추가 → 테스트 사용자에 본인 구글 계정 이메일 추가
4
OAuth 2.0 클라이언트 ID 발급
사용자 인증 정보 → 사용자 인증 정보 만들기 → OAuth 클라이언트 ID → 애플리케이션 유형: 웹 애플리케이션 → 승인된 JavaScript 출처에 본인 GitHub Pages 주소 입력
https://[내아이디].github.io
→ 만들기 → 클라이언트 ID 복사
5
gcal.js 파일에 Client ID 붙여넣기
gcal.js 파일 상단 CLIENT_ID: 'YOUR_GOOGLE_CLIENT_ID...' 부분을 복사한 클라이언트 ID로 교체 → GitHub에 업로드
6
연결 테스트
상단 Google 계정으로 연결 버튼 클릭 → 구글 계정 선택 → 권한 허용 → 연결 완료 후 배정 확정 시 자동으로 Google 캘린더에 일정이 추가됩니다 ✅

동기화 동작 방식
✅
배정 확정 시 자동 추가
배정 확정 버튼 클릭 즉시 담당자 Google 캘린더에 일정이 생성됩니다.
🔄
재배정 시 자동 수정
일시나 담당자가 바뀌면 기존 Google 캘린더 이벤트가 자동으로 업데이트됩니다.
❌
반려 시 자동 삭제
신청이 반려되면 Google 캘린더에서도 해당 일정이 삭제됩니다.
📨
담당자 초대 이메일
배정된 담당자 이메일로 Google 캘린더 초대장이 자동 발송됩니다.
⏰
자동 알림 설정
일정 1일 전 이메일, 1시간 전 팝업 알림이 자동으로 설정됩니다.
🗓️
전체 일괄 동기화
전체 동기화 버튼으로 모든 확정 일정을 한 번에 Google 캘린더에 반영합니다.
